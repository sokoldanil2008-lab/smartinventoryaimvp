# smartinventoryaimvp
1. Пітч (Pitch)
SmartInventory AI вирішує проблему критичних помилок та великих часових витрат при ручному обліку товарів на складах малих підприємств. Продукт дозволяє комірникам миттєво оцифровувати надходження товарів через ШІ-сканування накладних, забезпечуючи власникам бізнесу актуальну картину залишків у реальному часі без паперової рутини.
2. Ядро ідеї (MVP)
Мінімально життєздатний продукт зосереджений на одному ключовому сценарії: автоматичне розпізнавання документа та оновлення реєстру. Користувач повинен мати можливість зафіксувати запис про товар за допомогою камери, побачити цей запис у загальному цифровому списку та позначити його як «прийнятий», підтверджуючи коректність даних.
3. Карта стейкхолдерів (Stakeholder Map)
Цінність продукту безпосередньо сприйматимуть комірники та оператори складу, для яких спрощується щоденна робота та зникає потреба в ручному введенні. Досягнення бізнес-цілей та ефективність використання ресурсів оцінюватимуть власники бізнесу (замовники). На реалізацію проєкту через технічні обмеження ШІ-моделей та жорсткі часові рамки ітерації впливатиме команда розробників, яка забезпечує працездатність алгоритмів.
4. Мета першої ітерації
Створити робочий прототип системи, який дозволяє стабільно виконувати повний цикл «сканування-відображення-підтвердження» для друкованих накладних протягом тестового тижня.
5. Ознаки успіху (Definition of Success)
Автономність сценарію: Здатність нової людини успішно зафіксувати товар та побачити його у списку без сторонніх підказок чи інструкцій.
Стабільність операції: Відсутність критичних збоїв у процесі розпізнавання та збереження даних при щоденному використанні протягом тижня.
Лаконічність взаємодії: Виконання головного сценарію (від відкриття камери до появи запису в списку) займає не більше 10 секунд.
